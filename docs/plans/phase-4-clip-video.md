# Phase 4: クリップベース動画生成

ステータス: 未着手
規模: M
依存: Phase 2（steps.json v2 の t_start/t_end と overview）。Phase 3 と並行可

## 目的

解説動画を「静止画紙芝居」から「元録画のクリップ + ナレーション」に刷新し、操作の動きが見えるマニュアル動画にする。

## 現状の問題

- `videoGenerator.ts` はステップごとに静止画1枚を `-loop 1` でTTS長だけ表示して連結するのみ（C10）。アーティファクトに `t_start`/`t_end` があるのに未使用
- TTS失敗時に無音音声へサイレントフォールバックし、ユーザーが気づけない（NFR-4違反）
- ナレーション付き録画では、元音声（実演者の説明）を活かす選択肢がない

## 設計

### 1. クリップ切り出し
- 各ステップの `t_start`〜`t_end`（統合ステップは `source_segment_ids` の最小t_start〜最大t_end）で元録画から切り出し
- トリミング規則（env化、評価セットで較正）:
  - 前後パディング: 操作の文脈が見えるよう `CLIP_PAD_BEFORE_MS`（目安500ms）/ `CLIP_PAD_AFTER_MS`（目安800ms）
  - 長尺ステップ（タイピング合体等）は `CLIP_MAX_DURATION_S`（目安20s）で上限を設け、超過分は中間を倍速（`setpts`）または静止画ダイジェストに切替
- 切り出しは `-ss <start> -to <end> -i` の入力シークで高速化し、連結はセグメント再エンコード（解像度・fps・サンプルレート統一）後にconcat（既存方式踏襲）

### 2. 音声トラックの方針（ステップ単位で選択可能）
| モード | 内容 | 既定 |
|--------|------|------|
| `tts` | v2 narration をTTS合成し、クリップ音声をミュート | 無音録画の既定 |
| `original` | 元録画の該当区間音声をそのまま使用 | ナレーション付き録画の既定 |
| `mixed` | 元音声を減衰（ダッキング）してTTSを重畳 | オプション |
| `silent` | 無音 | narration 空のステップ |
- 既定モードは evidence.json の音声有無（transcript segments の有無）から自動選択し、UIでプロジェクト/ステップ単位の上書きを可能にする
- **クリップ長とTTS長の調停**（`tts` モード）: TTSがクリップより長い場合はクリップ末尾を静止フレームで延長（freeze frame）。短い場合はクリップ長に合わせ無音パディング

### 3. 構成要素の追加
- イントロ: `overview.task_title` + 概要ナレーションのタイトルカード（静止画生成、既存スライドレンダラーの表紙を流用可）
- ステップ字幕: 画面下部に `title` / `instruction` をdrawtextまたは焼き込み字幕で表示（env でON/OFF）
- アウトロ: `completion_criteria` のカード

### 4. フォールバックと可視化
- 元録画が取得できない・切り出し失敗時はステップ単位で従来の静止画+TTSにフォールバックし、`warnings` に記録
- TTS失敗時の無音フォールバックは**廃止せず可視化**: 生成結果に `audio_fallback: true` を記録し、UI完了通知で「N件のステップが無音になっています」と警告（NFR-4）

## タスク分解

1. クリップ切り出し+調停ロジック（音声モード `tts`/`silent` のみ、字幕なし）— 最小エンドツーエンド
2. `original`/`mixed` モード + 自動選択
3. イントロ/アウトロ/字幕 + フォールバック可視化のUI + **ステップの `t_start`/`t_end` 編集ルートとUI**（現行 `step.update` は title/operation/description/narration のみで t 編集は存在しない。クリップのカット調整の代替手段として本フェーズで新設する）

## やらないこと

- ズーム/パン等のカメラワーク演出（差分bboxズームは将来候補としてメモに留める）
- BGM
- 動画編集UI（カット調整はステップの t_start/t_end 編集で代替）

## 受け入れ基準

- [ ] 評価セットのナレーション付きケースで `original`、無音ケースで `tts` が自動選択され、エンドツーエンドで動画が生成される
- [ ] TTSとクリップの長さ調停（freeze frame / パディング）が動作する
- [ ] 音声・映像のズレが最終動画で許容内（手動確認項目。連結部のドリフトがないこと）
- [ ] クリップ取得失敗時に静止画フォールバックが効き、warningsに記録される
- [ ] 無音フォールバック発生がUIに表示される
- [ ] 10分録画からの動画生成が実用時間内（計測値を記録）

## リスク

| リスク | 対応 |
|--------|------|
| 元録画のアップロード原本が削除/欠損しているプロジェクト | 原本 `videoKey` の存在チェックを生成前に行い、欠損時は全編静止画モードへ自動切替 |
| クリップに無関係な操作・個人情報が映り込む | パディングを控えめな既定値にし、ステップ単位の t 調整UI（本フェーズのタスク3で新設）で対応 |
| ffmpegフィルタ複雑化によるデバッグ困難 | ステップごとに中間セグメントファイルを生成する現行方式を維持し、failしたステップを特定可能に保つ |
