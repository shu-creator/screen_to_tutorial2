# スライドデザイン分析

## 現在の問題点

### 1. 画像が大きすぎる
- 画像がスライドの大部分を占めている（y: 1.8, w: 9.0, h: 5.0）
- 説明文が画像の下に配置されているため、見づらい
- 画像の詳細が見えにくい（特に右側のパネルが切れている）

### 2. レイアウトの問題
- タイトル（y: 0.8）と画像（y: 1.8）の間隔が狭い
- 操作説明（y: 7.0）と詳細説明（y: 7.5）がスライドの下部に固まっている
- 説明文のフォントサイズが小さい（12-14pt）

### 3. 視認性の問題
- 画像が大きすぎて、説明文とのバランスが悪い
- 手順を説明する用途に向いていない

## 改善案

### レイアウト変更
1. **2カラムレイアウト**を採用
   - 左側: 画像（スクリーンショット）
   - 右側: 説明文（タイトル、操作、詳細）

2. **画像サイズを縮小**
   - 幅: 9.0 → 5.5
   - 高さ: 5.0 → 4.0
   - 位置: 左側に配置

3. **説明文を右側に配置**
   - タイトル: 大きく目立つように（28-32pt）
   - 操作: 中サイズ（18-20pt）
   - 詳細説明: 読みやすいサイズ（14-16pt）

4. **視覚的な改善**
   - 背景色を追加して、説明エリアを区別
   - アイコンや番号を追加して、視認性を向上
